### 用例：查看经营历程表

| 项       | 值                                        |
| ------- | ---------------------------------------- |
| ID      |                                          |
| 名称      | 查看经营历程表                                  |
| 创建者     | 陈俊达                                      |
| 最后一次更新者 | 陈俊达                                      |
| 创建日期    | 2017/9/14                                |
| 最后更新日期  | 2017/9/14                                |
| 参与者     | 财务人员，目的是查看经营历程表，可选进行红冲                   |
| 触发条件    | 财务人员需要查看某段时间内的经营历程表                      |
| 前置条件    | 财务人员已经登录。                                |
| 后置条件    | 如果财务人员进行了红冲，则保存红冲数据。                     |
| 优先级     | 高                                        |
| 正常流程    | 1. 财务人员进入查看经营历程表界面；<br/>2. 系统显示需要填写的信息；<br/>3. 财务人员填写筛选条件，包括 **时间区间，单据类型，客户，业务员，仓库**，单据类型分为**销售类单据（销售出货单，销售退货单）**、**进货类单据（进货单，进货退货单）**、**财务类单据（付款单，收款单，现金费用单）**、**库存类单据（报溢单，报损单，赠送单）**;<br/>4. 系统通过这些筛选条件，筛选出满足条件的所有单据，并显示给财务人员；<br/>5. 财务人员获得所需数据。 |
| 扩展流程    | 5a. 财务人员需要进行**红冲**：<br/>&emsp;1. 财务人员选定一条单据；<br/>&emsp;2. 系统生成一个一模一样但是把数量取负的单子并入账。<br/>5b. 财务人员需要进行**红冲并复制**：<br/>&emsp;1. 财务人员选定一条单据；<br/>&emsp;2. 系统生成一个一模一样但是把数量取负的单子，并显示给财务人员；<br/>&emsp;3. 财务人员进行修改，并提交；<br/>&emsp;4. 系统把新的单子入账。<br/>5c. 财务人员需要导出报表：<br/>&emsp; 1. 财务人员选中需要导出的报表和信息，并选定导出地址和格式；<br/>&emsp; 2. 系统根据财务人员的要求导出相应报表和信息。 |
| 特殊需求    | 财务人员不能修改和删除已经有的单据。                       |

| 扩展流程    | 2(2)a. 财务人员没有填写结束即提交：<br/>&emsp;1. 系统显示错误提示，并返回上一步。 |
| 特殊需求    | 无。                                       |
